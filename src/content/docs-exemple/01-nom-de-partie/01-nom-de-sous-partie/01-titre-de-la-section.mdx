## Les sections de la page

### L'en-tête (ou frontmatter)

Chaque page commence par une section d'en-tête, incluant les métadonnées et quelques éléments de configuration de la page
Cette section est encadrée par trois tirets `---`

On y trouve a minima ces deux lignes :

```
title : C'est le titre de la page tel qu'il apparaîtra, visible, en haut de page et dans la table des matières
description : utile pour les métadonnées, le référencement, etc.
```

Le paramètre `draft : true` permet de créer un brouillon de la page. On peut y travailler sans qu'elle soit mise en ligne en prod.
Une fois la page prête à être publiée, il n'y a qu'à supprimer cette ligne de paramétrage

### Les lignes d'import

Les lignes telles que

```
import { Tabs, TabItem } from "@astrojs/starlight/components";
import { YouTube } from "astro-embed";
```

permettent d'importer des modules ou composants extérieurs.
Ici, les composants `Tabs` et `TabItem` , des composants permettant de créer des tabulations sont importés du module `"@astrojs/starlight/components` et le composant `Youtube` est importé depuis `astro-embed`

## Composants

### Composants de Starlight

#### Composants à importer

[Page de la documentation officielle](https://starlight.astro.build/fr/components/using-components/)

Les composants de cartes `Card`, carte de liaison `LinkCard`, de groupe de cartes `CardGrid`, d'étapes `Steps`, de badges ou d'icônes doivent être importés depuis `@astrojs/starlight/components`

Ils sont ensuite utilisés selon les recommandations de la doc Starlight.

Par exemple, pour utiliser un composant d'étapes, on fera

```
import { Steps } from '@astrojs/starlight/components'; <- IMPORTATION

<Steps>      <- UTILISATION

1. On importe le composant en haut de document:


2. On entoure les éléments de la liste ordonnée avec `<Steps>`.

</Steps>
```

#### Notes et encarts

Starlight permet aussi de créer facilement des encarts (paragraphes colorés) avec la syntaxe suivante :

```
:::note
Ceci est une note d'information (sur fond bleu)
:::
```

```
:::note[Cette note a un titre !]
Ceci est une note d'information / astuce (sur fond violet) avec un titre personnalisé
:::
```

```
:::caution
Ceci est une note d'alerte (sur fond jaune, avec une icône "attention")
:::
```

```
:::danger[N'oubliez pas de sauvegarder]
Ceci est une note d'alerte de danger (sur fond rouge) à laquelle on a donné un titre
:::
```

### Composants additionnels

#### Intégration de vidéos

Le projet utilise [astro-embed](https://github.com/withastro/docs) pour intégrer facilement des vidéos de façon optimisée

- **Pour Youtube:**

  1. Importer le composant : `import { YouTube } from 'astro-embed';`
  2. L'utiliser à l'endroit souhaité en passant en paramètre l'ID de la vidéo ou son URL complet : `<YouTube id="TtRtkTzHVBU" />` ou `<YouTube id="https://youtu.be/TtRtkTzHVBU" />`

- **Pour Viméo:**

  1. Importer le composant : `import { Vimeo } from 'astro-embed';`
  2. L'utiliser à l'endroit souhaité en passant en paramètre l'ID de la vidéo ou son URL complet : `<Vimeo id="32001208" />` ou `<Vimeo id="https://vimeo.com/32001208" />`

## Organisation des pages de documentation

Les pages de documentation proprement dites sont dans le dossier `src/content/docs`.

Ce dossier est organisées en parties et sous parties numérotées pour maintenir un ordre logique dans la barre de navigation de gauche (table des matières).

Au sein de chaque dossier de sous-partie se trouve un dossier `images` contenant les images incluses dans les fichiers d'images présents dans la section.

Si vous ajoutez un sous-dossier ou une page de documentation, veillez à :

- Inclure ce dossier `images` si nécessaire afin de maintenir la structure du projet
- Ne pas modifier les noms de fichiers existants (risque de modification de l'URL et de liens cassés) mais d'intercaler vos ajouts à l'aide d'une numérotation mixte (par ex. `03a-validation-de-facture`)
